# 个人主页作品

## 作品名称
黄朝阳的个人主页

## 作品简介
这是一个现代化的个人主页网站，采用 HTML 和 CSS 技术构建。页面设计简洁美观，使用渐变色彩和响应式布局，展示个人信息、技能特长、兴趣爱好等内容。整体风格清新专业。

## 页面主要内容说明
| 模块 | 内容描述 |
|------|----------|
| 头部区域 | 个人头像、姓名、座右铭 |
| 信息卡片 | 学号、专业、电子邮件等基本信息 |
| 个人简介 | 详细介绍个人背景和技术能力 |
| 兴趣标签 | 展示个人兴趣爱好（编程、阅读、音乐） |
| 技能标签 | 展示技术技能（Java、Python、C++） |
| 联系方式 | 项目仓库链接等社交信息 |

## 本作业中你认为最能体现"软件作品"而不只是"单个页面"的一点
**CSS 变量系统与模块化设计**。通过 `:root` 定义统一的色彩变量、阴影效果和过渡动画，实现了：
- 全局样式统一管理，便于主题切换和维护
- 组件化标签设计（.tag、.info-card 等可复用组件）
- 响应式媒体查询适配不同设备屏幕
- 悬停交互效果增强用户体验

**分文件存储**
-  项目中的所有样式和布局都存储在 `style.css` 文件中，通过 `@import` 引入，使得代码结构清晰、可维护。

这些设计使项目具备了可扩展性和可维护性，符合软件工程的最佳实践。

- 简化了部分过度复杂的动画效果
- 手动添加了个人真实信息（学号、专业、邮箱等）
- 根据实际预览效果微调了 padding 和 margin 值
- 优化了中文内容的排版和换行

## 你认为 AI 最不可靠的一点是什么
**对具体业务逻辑和个性化需求的理解有限**。AI 生成的代码虽然语法正确、结构规范，但：
- 无法准确理解个人品牌风格的独特需求
- 生成的内容可能需要人工审核和调整
- 对于图片资源路径、真实个人信息等需要人工确认
- 某些交互细节需要实际测试后才能发现并修正

因此，AI 更适合作为辅助工具，最终决策和审核仍需开发者完成。
